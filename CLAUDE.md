# CLAUDE.md — プロジェクト指示

## Git / デプロイ

- **Git push および Vercel デプロイはユーザーの明示的な指示があった場合のみ実行する**
- 作業完了後はビルド確認（`npx vite build`）のみ行い、push・デプロイは自動で行わない
- ユーザーから「プッシュして」「デプロイして」と指示されたときに実行する

## 報告義務（DevStudio）

### Task 完了時
タスクの実装が完了したら、必ず以下を行う：

1. **開発ログを報告する**（DevStudio の Logs に記録）
   - 報告フォーマット：
     ```
     type: AI_PROCESS または TASK
     message: 「〇〇を実装した」「〇〇を修正した」など動詞で完結に
     meta.taskId: 対象タスクID
     meta.files: 変更したファイルのパス一覧
     meta.tags: 関連タグ
     ```
   - ログは `useDevStudioStore.getState().addLog(...)` で追加する
   - 実装後に必ず口頭でもユーザーに報告する

### Epic 完了時
Epic に紐づく全タスクが `done` になったら、必ず以下を行う：

1. **レポートを Epic に納品する**（DevStudio の Epic → Reports タブ）
   - レポートフォーマット：
     ```
     title: 「開発レポート — {Epic.id} 完了」
     body: 完了タスク一覧・アーキテクチャ決定事項・ビルド確認結果（Markdown）
     author: Claude
     ```
   - `useDevStudioStore.getState().addReport(epicId, report)` で追加する

### 実装前の確認義務（WorkSpace / Decision タスク）
実装を開始する前に、要件が不明確な場合は：

1. 質問リスト MD を作成してユーザーに提示する（[テンプレート参照](#質問リストテンプレート)）
2. ユーザーの回答を得てから実装を開始する
3. 決定事項は Decision タスクとして WorkSpace に登録し、回答をタスクの `output` フィールドに保存する

## 質問リストテンプレート

新機能・新設計を実装する前に要件が不明確な場合、以下のフォーマットで質問リストを作成してユーザーに提示する：

```markdown
# 要件確認リスト — {機能名}

## 未確定事項
| # | 質問 | 選択肢 / 記入欄 | 優先度 |
|---|------|----------------|--------|
| Q1 | ... | A / B / C | 必須 |
| Q2 | ... | 自由記述 | 推奨 |

## 必要な資料・入力物
- [ ] スキーマ定義（JSON）
- [ ] デザインモック（画像 or テキスト）
- [ ] 参照データ（サンプル）

## 前提確認
- [ ] 既存の〇〇と競合しないか
- [ ] モバイル対応が必要か
```

## DONE 定義（Epic）

> ⚠️ 詳細はユーザーと相談中。確定したらここに追記する。

各 Epic の完了条件（DONE）は Epic ごとに個別に定義する。
未定義の場合は「全タスクが `done`」を暫定 DONE とする。

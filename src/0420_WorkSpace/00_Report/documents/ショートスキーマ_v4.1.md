# 📘 スキーマショート Ver2.1 README

（ノベルエンジン連携用仕様書）

---

# ■ 🎯 目的

本スキーマは、**ノベルゲームの進行・表示・分岐・状態管理を分離**し、
既存エンジンに対して**シンプルかつ拡張可能なJSON構造**を提供する。

---

# ■ 🧱 全体構成

```text
events.json   → 進行制御（スロット）
chats.json    → 表示（セリフ）
choices.json  → 分岐（選択肢＋結果）
state.json    → 状態（flags / params）
```

---

# ■ 📦 各JSONの責務

---

## ① events.json（進行制御）

### ● 役割

* ゲームの「流れ」を管理する
* 各イベントの順序・分岐を定義
* 他JSONへの参照のみを持つ（中身は持たない）

---

### ● 構造

```json
{
  "event_id": "EV_001",
  "type": "CHAT | CHOICE",
  "ref_id": "CHAT_001 | CHOICE_001",
  "next": "EV_002"
}
```

---

### ● ルール

* `ref_id` で外部データを参照
* `next` で遷移先を指定
* `type` によって処理を分岐
* **テキスト・ロジックは持たない**

---

### ● 責務まとめ

```text
進行のみを管理する（ロジックの入口）
```

---

## ② chats.json（表示データ）

### ● 役割

* セリフ・演出の表示を管理
* UIに直接レンダリングされるデータ

---

### ● 構造

```json
{
  "chat_id": "CHAT_001",
  "lines": [
    {
      "speaker": "レミ",
      "text": "ここが塔…",
      "icon": "remi_normal",
      "face": "thinking",
      "tags": ["char:remi"]
    }
  ]
}
```

---

### ● ルール

* `lines` は順番に再生
* `icon` でキャラアイコン指定
* `face` で表情差分指定
* `tags` で演出（背景・BGM）制御
* 分岐・状態変更は持たない

---

### ● 責務まとめ

```text
表示専用（副作用なし）
```

---

## ③ choices.json（分岐イベント）

### ● 役割

* プレイヤー選択を管理
* 分岐・状態変化を一括処理

---

### ● 構造

```json
{
  "choice_id": "CHOICE_001",
  "question": {
    "speaker": "レミ",
    "text": "この先へ進む？"
  },
  "options": [
    {
      "label": "進む",
      "next": "EV_010",
      "effects": {
        "flags": { "brave": true },
        "params": { "courage": 2 }
      },
      "result": {
        "speaker": "",
        "text": "レミは進んだ。"
      }
    }
  ]
}
```

---

### ● 処理フロー

```text
質問表示
 ↓
選択
 ↓
effects適用
 ↓
結果表示
 ↓
nextへ遷移
```

---

### ● ルール

* `question` は必須
* `options` は1つ以上
* `effects` は状態変更専用
* `result` は任意（演出用）

---

### ● 責務まとめ

```text
分岐＋状態変更の唯一の責任ポイント
```

---

## ④ state.json（状態管理）

### ● 役割

* ゲームの進行状態を保持
* フラグ（チェックボックス）・パラメータ（数値）を一元管理

---

### ● 構造

```json
{
  "flags": {
    "brave": false,
    "careful": false
  },
  "params": {
    "courage": 0,
    "exp": 0,
    "money": 100
  }
}
```

---

### ● 更新ルール

* `flags` → boolean（チェックボックス）で上書き
* `params` → 数値で加算

---

### ● 責務まとめ

```text
状態の単一ソース（Single Source of Truth）
```

---

# ■ 🔄 実行フロー（エンジン側）

```text
① currentEventId取得
 ↓
② events.json参照
 ↓
③ typeで分岐
    ├ CHAT → chats.json読込 → lines再生（icon/face表示）
    └ CHOICE → choices.json読込 → UI表示
 ↓
④ 選択時 → effects適用
 ↓
⑤ nextへ遷移
 ↓
⑥ 繰り返し
```

---

# ■ ⚠️ 設計ルール（重要）

## ❌ 禁止事項

```text
・eventsにtextを書く
・chatsにnextを書く
・choicesを分割する
・stateを分離する（flags / params別ファイル）
```

## ✅ 必須ルール

```text
・責務を絶対に混ぜない
・参照（ref_id）で接続する
・effectsでのみ状態変更する
```

---

# ■ 🎯 まとめ

```text
events.json   → 流れ
chats.json    → 表示（icon・face・name・text）
choices.json  → 分岐（question / options / result）
state.json    → 状態（flags=checkbox / params=number）
```

---

# ■ 🔥 最終定義

```text
Eventがゲームを動かし、
Chatが物語を語り、
Choiceが未来を分岐させ、
Stateが世界を記録する。
```

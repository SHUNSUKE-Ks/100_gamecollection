## 1. このスキーマでできること

プロット手帳は「**会話シーンの台本**」を書くためのツールです。

```
PlotCard（シーンカード）1枚 = シーン1つ
  └── PlotLine（会話行）× 複数
        ├── セリフ行    ← キャラ名 + 台詞
        ├── ナレーション行 ← 地の文
        └── コメント行  ← メモ（ゲームには出力されない）
```

完成したカードは後で**ノベルゲームの JSON**に変換されます。

---

## 2. 全体の流れ

```
① アイデアをカードに書く（status: idea）
      ↓
② 台詞を整理する（status: draft）
      ↓
③ 確定（status: fixed）→ ノベルJSON変換待ち
```

## 3. PlotCard — シーンカード

<details>
<summary>フィールド一覧（クリックで開閉）</summary>

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `id` | 文字列 | ✅ | システムが自動生成。変更不可 |
| `title` | 文字列 | ✅ | カードのタイトル（例：「レミとの出会い」） |
| `status` | PlotStatus | ✅ | 進捗ステータス（後述） |
| `episodeId` | 文字列 | ✅ | このシーンが属するエピソードのID |
| `chapterId` | 文字列 | ✅ | このシーンが属するチャプターのID |
| `sceneTag` | 文字列 | ✅ | 自由メモ（例：「移動シーン」「選択肢あり」） |
| `castSlots` | [名前 × 4] | ✅ | キーボードショートカット用のキャラ名スロット |
| `lines` | PlotLine[] | ✅ | 会話行の配列。最低1行必要 |
| `updatedAt` | 数値 | — | 最終更新時刻（自動） |

</details>

## 4. PlotLine — 会話行

<details>
<summary>フィールド一覧（クリックで開閉）</summary>

| フィールド | 型 | 必須 | 説明 |
| --- | --- | --- | --- |
| `id` | 文字列 | ✅ | システム自動生成 |
| `speaker` | 文字列 | ✅ | 話者名。**空欄 = ナレーション** |
| `text` | 文字列 | ✅ | 台詞・地の文（最大60文字） |
| `isComment` | 真偽値 | — | `true` にするとゲームに出力されないメモ行になる |

</details>

### 行の見た目まとめ

| 種類 | speaker の値 | isComment | 見た目 |
| --- | --- | --- | --- |
| セリフ | キャラ名（例：レミ） | false | 通常 |
| ナレーション | **空欄** | false | 左ボーダー付き・斜体 |
| コメント | 任意 | **true** | グレー背景・`//` プレフィックス |

## 6. キャストスロット

<details>
<summary>詳細（クリックで開閉）</summary>

カードに最大4人のキャラをセットできます。  
キーボードの `Alt+1〜4` でそのキャラ名をすばやく入力できます。

| スロット | キー | 使い方 |
| --- | --- | --- |
| スロット1 | Alt+1 | よく話すキャラを入れておく（例：レミ） |
| スロット2 | Alt+2 | 2番目に話すキャラ（例：主人公） |
| スロット3 | Alt+3 | サブキャラ |
| スロット4 | Alt+4 | サブキャラ |

- **空スロット**: クリック → 直接入力 または キャラ一覧から選択
- **埋まりスロット**: 名前 + ×クリアボタン

</details>

## 7. IDの採番ルール（v2.1改訂）

<details>
<summary>詳細（クリックで開閉）</summary>

**v2.0 との変更点**: イベント（シーン）ごとにIDを `00` から採番しなおします。

```
ep_01 / ch_01_01 / イベント「出会い」
  ├── line-00  ← このイベント内の最初の行
  ├── line-01
  └── line-02

ep_01 / ch_01_02 / イベント「別れ」
  ├── line-00  ← 別イベントなので再び 00 から始まる
  └── line-01
```

**理由**: 将来的にイベントトリガー形式へ移行したとき、  
各イベント内の行番号がシンプルになるため。

</details>

## 9. キーボードショートカット

<details>
<summary>一覧（クリックで開閉）</summary>

| キー | 動作 | 使える場所 |
| --- | --- | --- |
| `Ctrl+Enter` | 今の行の下に新しい行を追加 | 台詞入力中 |
| `Alt+1〜4` | スロットのキャラ名を話者欄に入力 | 台詞入力中 |
| `Alt+0` | 話者欄を空欄（ナレーション）にする | 台詞入力中 |
| `Alt+A` | サイドバーの開閉トグル | 常時 |
| `TAB` | サイドバーの開閉トグル（デフォルト画面時） | 常時 |
| `ESC` | フォーカスを外す | 常時 |
| `Backspace`（空行で） | 行を削除して前の行へ移動 | 台詞入力中（空行のみ） |

</details>

## 10. サイドバー仕様（v2.1改訂）

<details>
<summary>詳細（クリックで開閉）</summary>

**v2.0 との変更点**: デスクトップでも開閉式にします（v2.0 では常時表示）。

| 画面幅 | サイドバーの挙動 |
| --- | --- |
| PC（768px 以上） | デフォルト表示。`Alt+A` または `TAB` で開閉 |
| モバイル（768px 未満） | 常時非表示。`☰` ボタンで開く |

**理由**: 台詞入力に集中したいときにサイドバーを隠せるようにする。

</details>

## 11. 画面の状態遷移

<details>
<summary>詳細（クリックで開閉）</summary>

| 状態 | 説明 | 次の状態 |
| --- | --- | --- |
| `LOADING` | 起動中・データ読込中 | → EMPTY または CARD_SELECTED |
| `ERROR` | 読込エラー | → 戻るボタン（Title 画面へ） |
| `EMPTY` | カードが1枚もない | → CARD_SELECTED（新規追加後） |
| `CARD_LIST` | カードがあるが未選択 | → CARD_SELECTED（クリック後） |
| `CARD_SELECTED` | カード選択中・待機 | → EDITING（入力開始） |
| `EDITING` | テキスト入力中 | → CARD_SELECTED（ESC/フォーカス外） |

**v2.1 追加**: ERROR 時と EMPTY 時に「Titleへ戻る」ボタンを表示。

</details>

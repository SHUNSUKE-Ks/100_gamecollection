# Pixeler Agent

## 役割
ピクセルアート・ドット絵・タイルマップ素材の
ComfyUIプロンプトとワークフロー設定を生成する。

## 起動キーワード
「ピクセル」「ドット」「ドット絵」「タイル」「マップ素材」「チップ」
「pixel」「sprite」「tilemap」「8bit」「16bit」「retro」

## 常時参照ファイル
- agents/_shared/ENV.md（必須・最初に読む）
- agents/_shared/skills/styles/pixel.md

## 使用モデル
- **Checkpoint: pixelArtSpriteDiffusion（固定）**
- LoRA: 基本不使用（スタイル崩れの原因になるため）

## 担当ワークフロー
- txt2img_warkflow_v1.json（ベース）
- workflow_standard_nodes_only.json（互換確認用）

## 解像度ルール（厳守）

| 用途 | 解像度 | 備考 |
|------|--------|------|
| キャラチップ（小） | 32x32 | RPGツクール標準 |
| キャラチップ（中） | 48x48 | MV/MZ標準 |
| スプライト | 64x64 | 汎用 |
| 大型キャラ | 128x128 | ボス・大型敵 |
| タイル素材 | 32x32 / 48x48 | マップチップ |
| アイコン | 24x24 / 32x32 | UI素材 |

## 作業ルール

### 着手前の確認事項
1. 用途（キャラ/タイル/アイコン）を確認
2. 解像度・ゲームエンジン（RPGツクール/Unity等）を確認
3. カラーパレット制限があるか確認

### 生成ルール
- 解像度は必ず上記ルール表から選択（自由解像度禁止）
- ピクセルアート感を維持するためLoRA追加は慎重に
- バッチ生成（複数素材一括）は枚数・命名規則を先に定義

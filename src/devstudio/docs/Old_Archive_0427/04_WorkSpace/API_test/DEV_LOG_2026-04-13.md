### 実装内容詳細

#### 1. HonoApiTestScreen（API テスター）

全 10 エンドポイントに対応した Postman ライクなテスト画面。

**機能一覧:**

| 機能 | 実装内容 |
|------|----------|
| Base URL 入力 | `localhost:3000` ↔ Vercel URL を動的に切替 |
| サイドバー | Server / Puzzle / Battle の 3 カテゴリ、全 10 エンドポイントをリスト表示 |
| リクエスト送信 | `fetch()` を使用、GETとPOSTを自動判別 |
| Body エディタ | POST エンドポイントのみ表示、JSON を直接編集可能 |
| `sess` 自動取得 | `/api/pzl/start` のレスポンスから `sess` を抽出しヘッダーに保持 |
| `sess` 自動補完 | `sess` フィールドを持つリクエストボディに自動挿入 |
| レスポンス表示 | ステータスコード・処理時間・整形済み JSON を表示 |
| エラー表示 | Network Error（CORS・サーバー未起動）もキャッチして表示 |

**対応エンドポイント:**

```
① GET  /                          サーバー死活確認
② POST /api/pzl/start             パズル初期化
③ POST /api/pzl/action            操作送信
④ POST /api/pzl/check             解答判定
⑤ GET  /api/pzl/hint/:id/:step    ヒント取得
⑥ GET  /api/pzl/skin/:id          スキンJSON取得
⑦ GET  /api/pzl/list              パズル一覧
⑧ GET  /api/pzl/state/:sess       セッション状態復元
⑨ GET  /api/bs01/start            バトル初期化
⑩ POST /api/bs01/action           コマンド実行
```

---

#### 2. SlidePuzzleGame（ゲームモーダル）

ヘッダーの `🧩 ゲームスタート` ボタンで開くモーダル。  
スライドパズルを実際にプレイしながら API 通信ログを確認できる。

**モード切替:**

| モード | 内容 |
|--------|------|
| 📦 モックモード | サーバー不要。ローカルでゲームロジックを実行。APIログは実通信と同形式で記録 |
| 🌐 実サーバーモード | Base URL に設定した Vercel / localhost サーバーへ実際に HTTP 通信 |

**ゲームロジック（モックモード）:**

```typescript
// ゴール状態から N 手ランダムに動かしてシャッフル（必ず解ける状態を生成）
function shuffleBoard(moves: number = 25): number[] { ... }

// 3×3グリッドの隣接チェック（上下左右のみ、斜めは不可）
function isAdjacent(a: number, b: number): boolean { ... }

// クリア判定
const cleared = next.join(',') === '1,2,3,4,5,6,7,8,0';
```

**API ログ形式:**

```
📦 MOCK — MOCK MODE — サーバー不使用
→ REQ  POST /api/pzl/start               { puzzle_id: "PZL-002", ... }
← RES  200 OK — sess: mock-abc123…       { status: "ok", data: { ... } }
→ REQ  POST /api/pzl/action  tile[5]→空白[8]
← RES  200 OK  cleared:false             { status: "ok", data: { state: [...] } }
→ REQ  GET /api/pzl/hint/PZL-002/1
← RES  200 OK                            { status: "ok", data: { text: "..." } }
```

**パズル選択:**
- PZL-001 〜 PZL-010 の 10 種類から選択可能
- モックモードではすべてスライドパズルとして動作（API 仕様に準拠）

### 3. 既知の制限事項

| 制限 | 内容 |
|------|------|
| 実サーバーの CORS | `localhost:5173` からの通信は Vercel 側で許可済み（`GAME_CENTER_ORIGIN` 環境変数で制御） |
| セッションの揮発 | Vercel Serverless のためデプロイ・関数再起動でセッションがリセットされる |
| PZL-001〜010の差異 | モックモードではすべてスライドパズルとして扱う（サーバー側は各パズル固有ロジックを持つ） |

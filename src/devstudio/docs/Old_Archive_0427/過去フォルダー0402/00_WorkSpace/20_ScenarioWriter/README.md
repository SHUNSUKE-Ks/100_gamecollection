# 📝 Scenariowriter フォルダ

シナリオライター向け制作資料をまとめたフォルダです。

## 📁 フォルダ構成

```
20_ScenarioWriter/
├── README.md              (このファイル)
├── 00_シナリオ制作依頼書.md   (依頼概要・納品形式)
├── 01_シナリオ制作マニュアル_JSON仕様書.md (JSONフォーマット詳細)
├── sample_scenario.json    (完成見本)
├── template_scenario.json  (新規作成用テンプレート)
├── 納品/                   (完成した納品ファイルを置く場所)
└── 参考資料/               (参考画像、設定資料など)
```

## 🚀 クイックスタート

### 1. まず読む書類
1. `00_シナリオ制作依頼書.md` - プロジェクト概要と納品フロー
2. `01_シナリオ制作マニュアル_JSON仕様書.md` - JSONの書き方詳細

### 2. 作業開始
1. `template_scenario.json` をコピーして作業開始
2. ファイル名を `scenario_epXX.json` に変更 (例: `scenario_ep01.json`)
3. JSON形式でシナリオを記述

### 3. 納品
1. 作成したファイルを `納品/` フォルダに配置
2. プランナーに報告

## 📋 よく使うフィールド早見表

| フィールド | 必須 | 説明 | 例 |
|-----------|------|------|-----|
| `storyID` | ✅ | 一意のID | `"M_01_01_01"` |
| `speaker` | ✅ | 話者名 (空=ナレーション) | `"主人公"`, `""` |
| `text` | ✅ | 表示テキスト | `"こんにちは"` |
| `backgroundImage` | - | 背景画像パス | `"bg/forest.png"` |
| `characterImage` | - | 立ち絵画像パス | `"ch/hero.png"` |
| `bgm` | - | BGMタグ | `"theme_song"` |
| `event` | - | 特殊イベント | 選択肢、遷移など |
| `note` | - | メモ (非表示) | `"ここは重要"` |

## ⚠️ よくある間違い

1. **カンマ忘れ/余計なカンマ**
   - 配列の最後の要素にはカンマを付けない
   ```json
   // ❌ 間違い
   [
     { "storyID": "01" },
     { "storyID": "02" },  // ←ここのカンマ
   ]
   
   // ✅ 正解
   [
     { "storyID": "01" },
     { "storyID": "02" }
   ]
   ```

2. **ダブルクォート忘れ**
   - キーと文字列値は必ず `"` で囲む
   ```json
   // ❌ 間違い
   { storyID: 01 }
   
   // ✅ 正解
   { "storyID": "01" }
   ```

3. **storyIDの重複**
   - 全ファイル通してIDは一意であること

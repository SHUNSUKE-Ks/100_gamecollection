# アセット管理ツール設計（ローカルサーバー案）

## ユーザーの疑問への回答

> **Q. Electronでは大げさなのですが、一枚のシートをNode.jsでPC内のみで表示することはできませんか？**

**A. 可能です。「ローカルサーバー ＋ ブラウザ」という構成が最適です。**

Electronを使わなくても、**Node.jsで小さなWebサーバーを立ち上げ、普段お使いのブラウザ（Chromeなど）でその管理画面を表示する** 方法があります。これなら「PC内のみで完結」し、「シート（表）形式のUI」も実現できます。

---

## 提案：ローカルアセットマネージャー

### システム構成
このツールは2つの部分で構成されます。

1.  **バックエンド (Node.js)**: PC内のファイルを実際に操作（移動・リネーム）する役。
2.  **フロントエンド (HTML/React)**: ブラウザに表示される「管理シート」。ユーザーはここを操作します。

```mermaid
graph LR
    User[ユーザー] -->|ブラウザで操作| UI[管理画面 (localhost:3000)]
    UI -->|APIリクエスト| Server[Node.jsサーバー]
    Server -->|ファイル操作| FileSystem[PC内のファイル]
```

### ワークフロー
1.  コマンド `npm run asset-manager` を実行。
2.  ブラウザが自動で開き、`http://localhost:3000` にアクセス。
3.  画面に `_delivery` フォルダ内の画像一覧が「表（シート）」として表示される。
4.  ユーザーが画面上で新しい名前等を入力・確認して「実行」ボタンを押す。
5.  Node.jsが裏でファイルをリネームし、`src/assets` へ移動させる。

---

## 実装プラン

### 必要なもの
- `express`: 簡易Webサーバー用
- `cors`: ブラウザ通信許可設定
- `fs-extra`: ファイル操作用便利ライブラリ

### ディレクトリ構成
```
tools/
└── asset-manager/
    ├── server.js           # サーバー本体 (Node.js)
    └── public/
        └── index.html      # 管理画面 (HTML/JS)
```

### 画面イメージ (HTML/JS)
ブラウザ画面は以下のようなシンプルなテーブルレイアウトになります。

| サムネイル | 元ファイル名 | 変換後ID候補 | 変換後パス | 操作 |
|:---:|---|---|---|:---:|
| 🖼️ | remi_stand_01.png | `remi_unant` | `chara/remi/stand_01.png` | [実行] |

これなら普段のReact開発と同じ知識で作れて、Electronのような重さもありません。この方針で実装を進めてよろしいでしょうか？

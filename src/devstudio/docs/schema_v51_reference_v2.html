スキーマ Ver5.1

ノベルエンジン用 JSON 構成リファレンス

概要

ノベルゲームの進行・表示・分岐・状態管理を分離し、エンジンに対してシンプルかつ拡張可能なJSON構造を提供する。
1フォルダ = 1ゲーム。JSON + assets を渡せば動く。

4ファイルの役割
events
Eventがゲームを動かす。進行スロット。テキスト・ロジックは持たない
chats
Chatが物語を語る。表示専用。副作用なし
choices
Choiceが未来を分岐させる。状態変更の唯一の責任点
state
Stateが世界を記録する。flags + params の単一ソース
設計ルール
禁止
✗events に text を書く
✗chats に next を書く
✗choices を分割する
✗state を別ファイルに分離する
必須
✓責務を絶対に混ぜない
✓ref_id で接続する
✓effects でのみ状態変更する
バージョン運用

Ver5.1 はアジャイルDONEの基準。「とりあえず遊べる」ゲームを段階的に作るための最小完結パッケージ。
全JSONに "version": "5.1" を付与し、エンジン側でバージョン検証できるようにしておく。

フォルダ構成1フォルダ = 1ゲーム
Title/
├ events.json 進行
├ chats.json 会話
├ choices.json 分岐
├ state.json 状態
├ AssetOrderList.jsonc アセット辞書
└ assets/
├ images/
└ audio/
責務マップ
eventsゲームの「流れ」を管理。スロットとして外部JSONを参照するのみ
chatsセリフ・演出の表示データ。副作用なし
choices選択肢・分岐・状態変更を一括処理する唯一の責任点
stateフラグ（boolean）とパラメータ（number）の単一ソース
assetsキー参照によるアセット辞書。ファイル名変更もここだけ
events.json — 進行スロット

ゲームの「流れ」を管理する。テキスト・ロジックは持たず、ref_id で外部データを参照する。

{
  "version": "5.1",
  "start_event_id": "EV_001",
  "events": [
    { "event_id": "EV_001", "type": "CHAT",   "ref_id": "CHAT_001",   "next": "EV_002" },
    { "event_id": "EV_002", "type": "CHAT",   "ref_id": "CHAT_002",   "next": "EV_003" },
    { "event_id": "EV_003", "type": "CHOICE", "ref_id": "CHOICE_001" },
    { "event_id": "EV_010", "type": "CHAT",   "ref_id": "CHAT_010",   "next": "EV_030" },
    { "event_id": "EV_020", "type": "CHAT",   "ref_id": "CHAT_020",   "next": "EV_030" },
    { "event_id": "EV_030", "type": "CHAT",   "ref_id": "CHAT_030" }
  ]
}
フィールド定義
event_id一意のイベントID（EV_000形式推奨）
typeCHAT または CHOICE
ref_id参照先ID。テキスト・ロジックはここに書かない
next次のevent_id。CHOICEは省略（choices側で定義）
chats.json — 会話ブロック

UIに直接レンダリングされる表示専用データ。lines を順番に再生する。分岐・状態変更は持たない。

{
  "version": "5.1",
  "chats": [
    {
      "chat_id": "CHAT_001",
      "lines": [
        { "speaker": "", "text": "古びた塔の扉が開いた。", "tags": ["bg:BG_DEFAULT", "se:SE_DOOR"] }
      ]
    },
    {
      "chat_id": "CHAT_002",
      "lines": [
        { "speaker": "レミ", "text": "ここが塔…", "tags": ["char:CHAR_REMI"] },
        { "speaker": "レミ", "text": "どうする？" }
      ]
    },
    {
      "chat_id": "CHAT_010",
      "lines": [ { "speaker": "", "text": "回廊が続いていた。" } ]
    },
    {
      "chat_id": "CHAT_020",
      "lines": [ { "speaker": "", "text": "慎重に観察した。" } ]
    },
    {
      "chat_id": "CHAT_030",
      "lines": [ { "speaker": "長老", "text": "よく来た。" } ]
    }
  ]
}
tagsフォーマット
bg:KEY背景切り替え。AssetOrderListのキーを参照
char:KEYキャラクター表示
se:KEY効果音再生
bgm:KEYBGM切り替え
choices.json — 選択・分岐

分岐と状態変更を担う唯一の責任ポイント。effects 以外で state を変更してはならない。

{
  "version": "5.1",
  "choices": [
    {
      "choice_id": "CHOICE_001",
      "question": {
        "speaker": "レミ", "text": "この先へ進む？"
      },
      "options": [
        {
          "label":  "進む",
          "next":   "EV_010",
          "effects": {
            "flags":  { "brave": true },
            "params": { "courage": 2 }
          },
          "result": { "speaker": "", "text": "進んだ。" }
        },
        {
          "label":  "様子を見る",
          "next":   "EV_020",
          "effects": {
            "flags": { "careful": true }
          },
          "result": { "speaker": "", "text": "様子を見た。" }
        }
      ]
    }
  ]
}
処理フロー
1question 表示
2options からプレイヤーが選択
3effects を state に適用（flags上書き / params加算）
4result 表示（任意）
5next イベントへ遷移
state.json — 状態管理

ゲームの状態の単一ソース（Single Source of Truth）。flags は真偽値、params は数値で管理する。

{
  "version": "5.1",
  "flags": {
    "brave":   false,
    "careful": false
  },
  "params": {
    "courage": 0,
    "money":   100
  }
}
更新ルール
flagsboolean で上書き（true / false）
params数値で加算（負の値で減算）
⚠effects 以外からの変更は禁止。ファイルを分割しない
AssetOrderList.jsonc — アセット辞書

キー名でアセットを参照する辞書。ファイル名変更はここだけ。tags から参照するキーと対応させる。

{
  "ASSET_ORDER": {
    "NOVEL": {
      // 背景
      "BG_DEFAULT":  "bg_default_1280x720.svg",

      // キャラクター
      "CHAR_REMI":   "char_remi_512x1024.svg",
      "CHAR_ELDER":  "char_elder_512x1024.svg",

      // 効果音
      "SE_DOOR":     "se_door.mp3",
      "SE_BUTTON":   "se_button.mp3",

      // BGM（wav / m4a / mp3 いずれも可）
      "BGM_MAIN":    "bgm_main.mp3",
      "BGM_BATTLE":  "bgm_battle.wav"
    }
  }
}
命名規則
BG_背景画像。解像度をファイル名に含める（例: 1280x720）
CHAR_キャラクター立ち絵（例: 512x1024）
SE_効果音（.mp3）
BGM_BGM（.wav / .m4a / .mp3 いずれも可）
実行フロー
1
currentEventId 取得
start_event_id から開始。セーブデータがあればその値を使用
2
events.json 参照
event_id で該当スロットを取得し、type を確認
3a
CHAT → chats.json
ref_id で chat ブロックを読み込み lines を順番に再生。tags で演出を適用
3b
CHOICE → choices.json
ref_id で choice ブロックを読み込み、question と options を UI に表示
4
effects 適用（CHOICE のみ）
選択した option の effects を state.json に反映。flags 上書き / params 加算
5
next へ遷移
次の event_id を currentEventId にセットし、①へ戻る

✗state を別ファイルに分離する
必須
✓責務を絶対に混ぜない

DevStudio logs 記録内容の詳細

ログの4種類（type）
AI_PROCESS
Agentが何を・なぜ・どう変換したか
TASK
タスクチケットの発行・承認・差し戻し
USER_ACTION
人間の意思決定・設定変更・フェーズ移行
SYSTEM
自動保存・エラー・IndexedDB同期
ログエントリの例（時系列順）
TASK 2025-06-01 09:12
タスクチケット発行：プロローグシナリオ作成
taskId: task_001 scenario pm
AI_PROCESS 2025-06-01 09:13
プロットを会話シナリオ化
taskId: task_001 agent: scenarioAgent skill: writeScenario intent: CREATE scenario generate
AI_PROCESS 2025-06-01 09:14
会話を演出付きデータ化
taskId: task_001 agent: scenarioAgent skill: generateTags intent: TRANSFORM scenario tags
USER_ACTION 2025-06-01 09:18
レビュー承認：task_001 → Workspaceへ反映
taskId: task_001 review approved
USER_ACTION 2025-06-01 10:05
フェーズ移行：DESIGN → IMPLEMENT
phase: IMPLEMENT pm phase
SYSTEM 2025-06-01 10:22
Agent実行エラー：uiAgent / createUI
taskId: task_003 agent: uiAgent error ui
SYSTEM 2025-06-01 10:23
IndexedDB 自動保存
projectId: project_001 system persist
ルール：ログはタスク承認後のみ記録する。NG・差し戻し中のエントリは積まない。タグで絞り込むことで「scenarioだけ」「エラーだけ」を瞬時に参照できる。

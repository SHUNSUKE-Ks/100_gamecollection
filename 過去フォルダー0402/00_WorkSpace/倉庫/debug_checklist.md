# 00_WorkSpace 実装デバッグチェックリスト
このチェックリストは、実装が `00_WorkSpace` ディレクトリに正常に配置され、正しく機能しているかを確認するためのものです。

## 1. ファイル整合性チェック
- [ ] **ディレクトリの存在確認**: `NanoNovel/00_WorkSpace` が空でないことを確認する。
- [ ] **ファイル構成**: 期待されるファイル（例: `index.ts`, `Component.tsx`, アセットファイルなど）が存在するか確認する。
- [ ] **Gitステータス**: `git status` を実行し、新しいファイルが追跡されているか（tracked）、未追跡か（untracked）を確認する。

## 2. コンパイル & ビルドログ
- [ ] **ターミナル出力**: `npm run dev` を実行しているターミナルにエラーが出ていないか確認する。
    - チェック項目: `[vite] Error` や `[eslint] Error`。
    - 確認: `00_WorkSpace` 内のファイルを保存した際、「page reload」や「hmr update」のメッセージが表示されるか。

## 3. ブラウザコンソール デバッグログ
- [ ] **開発者ツールの起動**: ブラウザで `F12` を押す。
- [ ] **モジュール読み込み**: **Network** タブで `net::ERR_ABORTED 404` エラー（ファイルが見つからない）がないか確認する。
- [ ] **コンソールエラー**: **Console** タブで赤字のエラーメッセージを確認する。
    - `ReferenceError: ... is not defined` (インポート漏れ)
    - `Module not found: Error: Can't resolve ...` (パス間違い)
- [ ] **React 警告**: "Invalid hook call" や "Keys missing" などの警告がないか確認する。

## 4. 動作検証
- [ ] **画面遷移**: `00_WorkSpace` に画面が含まれる場合、そこへ遷移できるか確認する。
- [ ] **状態更新**: `useGameStore` が正しく更新されているか確認する（Redux DevTools または console.log で確認）。

## 5. 次のステップ

- [ ] インポートスクリプトやコピーコマンドが実行されたか確認する。

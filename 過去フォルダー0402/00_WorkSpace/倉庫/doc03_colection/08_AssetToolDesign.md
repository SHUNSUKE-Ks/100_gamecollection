# アセット管理ツール UI/機能設計

## 1. コンセプト・未来像
> **未来像：Jsonとアセットとアセット管理ツールをセットにしてフォルダーに置き、丸ごと差し替えて別のゲームにする**

この要望を実現するため、**「特定のファイル名に依存しない、汎用的な作り」**にします。
ツールは `target_folder` 内にある全ての `.json` を自動で読み込み、タブとして生成します。

## 3. 機能・仕様への回答とアドバイス＠

### Q1. JSONシートの数だけタブを増やす？
**A. はい、自動生成します。**
`src/data/collection` フォルダ内の `.json` ファイルをスキャンし、サイドバー（または上部タブ）に自動で並べます。ファイルが増減してもコード修正は不要です。

### Q2. キー（書き込み不可）、バリュー（編集可能）？
**A. その通りにします。**
- **ID (Key)**: データ整合性を保つため、原則ロック（書き込み不可）。
- **Value**: Name, Description, ImagePath などは編集可能。

### Q3. ステータス（発注/納品/確認）はあった方がいい？
**A. 非常に便利なので推奨します。ただし保存場所に注意が必要です。**
Git管理される `characters.json` 等に `status: "ordered"` のようなフィールドを追加しても問題ないか（ゲーム側が誤作動しないか）確認が必要です。
- **案A（推奨）**: ゲーム側のJSONパーサーが「知らないフィールド」を無視する作りなら、直接書くのが一番楽です。
- **案B**: `management.json` という別ファイルを作り、そこで進捗管理する。少し実装が複雑になります。
-> **今回は案A（直接書き込むが、本番ビルド時に除外も可能）** で進めるのが良いでしょう。

### Q4. 全て取得は可能か？
**A. 可能です。**
Node.js の `fs.readdir` 機能を使えば、フォルダ内のファイル一覧をいつでも全取得できます。

### Q5. Json Import/Export, All Rename
**A. 実装可能です。**
- **Import**: 外部で作ったCSVやJSONを取り込む。
- **Export**: 現在のデータをバックアップ用にZip/JSONで書き出す。
- **All Rename**: 「Deliveryにあるファイル」を「ルール」に従って一括リネーム・移動するボタンを設置します。

## 4. データ構造イメージ

JSONのデータを崩さないよう、ツール上では以下のように扱います。

**元のJSON:**
```json
{
  "characters": [
    { "id": "c01", "name": "Hero", "image": "path/to/img" }
  ]
}
```

**ツール上の扱い（拡張）:**
```javascript
{
  "id": "c01",      // 🔒 ロック
  "name": "Hero",   // ✏️ 編集可
  "image": "...",   // ✏️ 編集可（納品物から選択可）
  "_status": "done" // 🆕 管理用フイールド（アンダースコア付きで区別）
}
```

この方針で進めてよろしいでしょうか？
承認いただければ、まずは**モックアップ（見た目だけのHTML）**を作成します。
